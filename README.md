# tiny-db — SQL-база данных с нуля + веб-админка tinydb Studio

Реляционная база данных, написанная **полностью с нуля на TypeScript, без внешних
библиотек в ядре** — свой формат хранения на диске, B+-деревья, собственный SQL
(лексер, парсер, планировщик, исполнитель), транзакции с WAL и блокировками. Поверх
движка — графическая веб-админка **tinydb Studio** (мини-pgAdmin в браузере).

## Зачем этот проект

Цель — показать, как устроена настоящая СУБД изнутри, собрав её своими руками, без
готовых решений. В ядре **нет ORM, готовых баз, сторонних SQL-парсеров и прочих
библиотек** — только TypeScript и стандартная библиотека Node.js. Из зависимостей
лишь dev-инструменты для запуска и проверки типов (`tsx`, `typescript`,
`@types/node`). Это учебный / портфолио-проект, а не production-СУБД.

## Возможности

**Движок:**
- Хранение на диске: append-only лог с восстановлением (replay), компакция,
  durability через `fsync`, устойчивость к обрыву записи.
- B+-деревья на страницах по 4 КБ: поиск/вставка/удаление за O(log n),
  упорядоченный обход по связанным листьям.
- Таблицы, схемы и типы (`INTEGER`, `TEXT`), order-preserving кодировка ключей.
- Свой SQL: лексер → парсер (рекурсивный спуск) → AST → исполнитель с планировщиком.
- Вторичные индексы (B-дерево на колонку) и выбор плана (индекс vs полный скан).
- Транзакции: `BEGIN`/`COMMIT`/`ROLLBACK`, WAL с redo-восстановлением после сбоя,
  атомарность по нескольким файлам.
- Конкурентность: строгий 2PL, блокировки таблиц S/X без deadlock (no-wait).
- `JOIN` (эквиджойн), агрегаты (`COUNT`/`SUM`/`MIN`/`MAX`/`AVG`), `GROUP BY`.
- Полный CRUD: `INSERT` / `SELECT` / `UPDATE` / `DELETE`, удаление таблиц.
- **314 автотестов** (свой мини-раннер, без jest/vitest).

**tinydb Studio (веб-админка):**
- Вход по логину/паролю, сессия в cookie.
- Несколько баз данных, создание и удаление из интерфейса.
- Редактор SQL с подсветкой синтаксиса и выполнением запросов.
- Просмотр данных в таблице; правка ячеек, добавление и удаление строк;
  удаление таблиц.
- Вкладка «Схема»: таблицы, колонки, типы, первичные ключи.
- Импорт SQL-скрипта из `.sql`-файла и экспорт результата в CSV / JSON.
- История запросов со сворачиванием дублей и очисткой.
- Переключение языка интерфейса RU / EN.

## Требования

- **Node.js 20+**
- npm

## Установка и запуск

```bash
git clone https://github.com/y6861977-ui/tinydb-studio.git
cd tinydb-studio
npm install
```

Проверка сборки и тестов:

```bash
npm test           # прогнать все тесты
npm run typecheck  # проверка типов
```

**SQL-консоль в терминале** (данные лежат в указанной папке):

```bash
npm run repl mydata
```

**Веб-админка tinydb Studio:**

```bash
npm run studio <папка-с-базами> <порт>
# пример:
npm run studio studio-data 8137
```

Затем откройте <http://localhost:8137>. Логин и пароль по умолчанию — `admin` /
`admin` (поменяйте через переменные окружения, см. ниже). Остановить сервер —
`Ctrl+C` в терминале, где он запущен.

### Конфигурация (переменные окружения)

| Переменная     | Назначение                                   | По умолчанию |
|----------------|----------------------------------------------|--------------|
| `STUDIO_USER`  | логин для входа в Studio                      | `admin`      |
| `STUDIO_PASS`  | пароль для входа в Studio                      | `admin`      |
| `STUDIO_HOST`  | адрес прослушивания                            | `127.0.0.1`  |
| `PORT`         | порт (если не передан аргументом)              | `8080`       |

Пример с заданными логином/паролем:

```bash
STUDIO_USER=me STUDIO_PASS='секрет' npm run studio studio-data 8137
```

## Поддерживаемый SQL

`CREATE TABLE`, `CREATE INDEX`, `INSERT`, `SELECT` (`*` / список колонок /
агрегаты, `JOIN ... ON`, `WHERE` с `AND`/`OR`/скобками и операторами
`= != <> < > <= >=`, `GROUP BY`), `UPDATE ... SET ... WHERE`,
`DELETE ... WHERE`, `BEGIN` / `COMMIT` / `ROLLBACK`.

Пока не поддерживается: `ORDER BY`, `LIMIT`, `LEFT JOIN`, `DROP` в SQL,
`NULL`, `UNIQUE`, `IN` / `BETWEEN`. Типы колонок — только `INTEGER` и `TEXT`.

## Структура проекта

```
src/
  db.ts         append-only лог-хранилище (ключ-значение) + replay + компакция
  record.ts     бинарный формат записи лога
  pager.ts      страницы 4 КБ на диске, позиционное I/O
  node.ts       сериализация узла B+-дерева
  btree.ts      B+-дерево на диске
  row.ts        сериализация строки и order-preserving кодировка ключа
  types.ts      система типов и валидация схемы
  table.ts      таблица поверх B+-дерева + вторичные индексы
  locks.ts      менеджер блокировок (S/X, no-wait)
  wal.ts        write-ahead log и транзакции
  database.ts   таблицы, каталог схем/индексов, транзакции
  sql/          лексер, парсер (AST) и исполнитель SQL
  sql-repl.ts   SQL-консоль в терминале
  server.ts     HTTP-сервер tinydb Studio (node:http, без Express)
web/
  tinydb-studio.html   вся веб-админка (HTML + CSS + JS в одном файле)
tools/
  inspect-log.ts, inspect-tree.ts   просмотр данных на диске
test/
  *.test.ts + мини-раннер
```

Осмотреть данные на диске:

```bash
npm run inspect:log  <файл-лога>
npm run inspect:tree <файл>.tbl
```

## Безопасность

Инструмент рассчитан на локальное использование. Сделано по умолчанию:

- Сервер слушает **только `127.0.0.1`** — из локальной сети он не виден. Открыть
  наружу можно осознанно через `STUDIO_HOST=0.0.0.0`, но тогда **обязательно
  смените логин и пароль**, иначе любой в сети войдёт по `admin/admin`.
- Вход: сравнение пароля за постоянное время (защита от timing-атак) и
  анти-брутфорс (10 неудачных попыток с адреса → блокировка на минуту).
- Сессии живут 8 часов; cookie — `HttpOnly`, `SameSite=Strict`.
- Заголовки безопасности (`Content-Security-Policy`, `X-Content-Type-Options`,
  `X-Frame-Options`), ограничение размера тела запроса.
- Имена таблиц, колонок и баз строго ограничены (`[A-Za-z_][A-Za-z0-9_]*` и
  подобное), значения экранируются — SQL-инъекции и обход путей исключены.
